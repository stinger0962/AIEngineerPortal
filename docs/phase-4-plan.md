# Phase 4 Plan

## Goal

Turn the learning side of the portal into real curriculum instead of short seed notes.

## Definition of Done

Phase 4 is complete when the portal provides durable, high-quality learning material across:

- learning paths
- course tracks
- coding practice
- knowledge references
- interview prep tied back to actual lessons and projects

The content should be worth returning to for daily study, not just enough to make the UI look populated.

## Execution Order

### Batch A: Python Foundation Rewrite

- Rewrite the full `Python for AI Engineers` path
- Upgrade the lesson page rendering experience so long-form lessons are readable
- Align lesson time estimates with actual content depth
- Add verification for richer lesson content

### Batch B: First Full Course Track

- Expand `AI Engineer Foundations` into a coherent multi-week sequence
- Tie milestones to specific paths, exercises, and project checkpoints

### Batch C: Practice Deepening

- Add hints, review notes, and stronger explanation content to exercises
- Introduce progression within categories so drills build on each other

### Batch D: Knowledge Reference Upgrade

- Rewrite the first durable reference pack around:
  - Python for AI backends
  - LLM app foundations
  - retrieval
  - evaluation

### Batch E: Interview and Project Alignment

- Rework interview question packs to match rewritten curriculum
- Expand project blueprints so lessons turn into visible portfolio proof

## Phase 4 Checkpoints

### Checkpoint 1

- `Python for AI Engineers` path is fully rewritten
- Lesson renderer supports structured long-form content cleanly

### Checkpoint 2

- `AI Engineer Foundations` course becomes a real guided track
- Practice system feels progression-based rather than flat

### Checkpoint 3

- Knowledge, interview, and project content are aligned with the rewritten curriculum
- Phase 4 checkpoint doc and live verification are complete

## Risks To Watch

- Overwriting user progress while syncing content
- Inflating lesson durations without adding enough material
- Adding too much AI-generated filler instead of durable, reviewed material

## Working Principle

Every content change in Phase 4 should answer one question:

Would this genuinely help the user spend a focused study session making real progress toward becoming an AI engineer?
