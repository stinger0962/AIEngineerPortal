# 紫微斗数 3D 交互算命系统 · 设计文档

日期：2026-06-09
状态：已与用户逐段确认
定位：AIEngineerPortal 门户的新 sidebar 板块（路由 `/ziwei`），非独立项目

## 1. 概述

一个 3D 沉浸式紫微斗数系统：用户为自己和家人朋友建立命主档案，在悬浮于星空中的 3D 命盘上探索本命盘、大限流年与合盘配对，由具备长期记忆的 AI 解盘师陪伴解读。知识底座来自开源仓库 [Renhuai123/ziwei-doushu](https://github.com/Renhuai123/ziwei-doushu)（MIT，需标注来源）的格局规则库与古籍原文。

**设计总原则：用户体验优先**——排盘零延迟、动画可跳过、加载有骨架、移动端可用、不懂紫微的人也能看懂（术语随手白话解释）。

## 3. 视觉与交互设计

### 3.1 三层空间（一镜到底）

1. **环境层（永久）**：星云 + 闪烁星点的宇宙舞台，所有元素悬浮其中。
2. **主盘层**：经典 4×4 紫微方盘 3D 化——发光悬浮罗盘，可拖动旋转/倾斜俯瞰；星曜为宫位上方漂浮光球；四化为宫与宫之间飞渡的光束；中宫显示生辰与五行局。懂紫微者零学习成本。
3. **单宫内景层**：点击任意宫位，镜头一镜到底俯冲飞入，该宫星曜化作环形殿牌环绕中央旋转，殿牌展示星曜亮度、四化标记；环阵旁悬浮该宫格局解读卡。返回即镜头拉回主盘。

### 3.2 AI 镜头联动

AI 解盘师通过结构化指令反向控制镜头：说到哪一宫，镜头飞到哪一宫；讲流年时时间轴高亮对应年份。指令集（Claude tool_use）：

- `focus_palace(palace)` — 飞入指定宫位内景
- `overview()` — 拉回主盘总览
- `highlight_stars(stars[])` — 高亮指定星曜光球
- `show_horoscope(year)` — 切流年盘并定位时间轴
- `explain_term(term)` — 前端弹出术语白话卡片

### 3.3 对话坞三态与回看

- **收起态**：右下角发光 ✦ 小球。
- **常规态**：浮动坞，最近消息 + 输入框 + 人设切换。
- **展开态**：右侧滑出全高玻璃态面板，滚动回看本次会话全部消息；顶部「历史会话」按场景分组列出该档案过往对话，可完整回看并「从这里继续聊」。

### 3.4 页面控件

- 左上：档案切换 chip（👤 名字 ▾），下拉建档/切换。
- 顶部居中：场景 pills——本命盘 / 流年 / 合盘 / 报告。
- 底部居中（流年场景）：年份时间轴，拖动切流年，大限分段显示。
- 「报告」tab：每次深度解读生成的结构化报告存档列表，随时回看。

### 3.5 移动端

响应式降级：对话坞变为底部抽屉（上滑展开）；3D 保留但降低粒子/后期效果档位；触控手势（单指旋转、双指缩放、点按飞入）。

## 4. 系统架构

```
frontend/src/app/ziwei/page.tsx          ← 主页面（动态 import 3D，按路由隔离 bundle）
frontend/src/components/ziwei/
  Scene3D/        ← R3F：环境层、主盘层、单宫内景、镜头控制器、指令执行器
  ChartPanel/     ← 档案管理、生辰输入（公历/农历+时辰）、人设切换
  OracleChat/     ← 对话坞三态、历史会话、流式渲染
frontend/src/lib/ziwei/                   ← iztro 封装、chart_json 标准化、格局检测（前端镜像）

backend/app/api/v1/routes/ziwei.py        ← 档案 CRUD、对话、记忆、知识查询、报告
backend/app/services/ziwei_service.py     ← 业务逻辑、格局规则引擎
backend/app/services/ziwei_oracle.py      ← Claude 解盘循环（仿 copilot_loop 模式）
backend/app/data/ziwei_knowledge/         ← patterns.json / stars.json / sihua.json
                                            classics.json / personas.json
```

**3D 技术栈**：react-three-fiber + @react-three/drei + @react-three/postprocessing（Bloom 辉光、Vignette）。排盘引擎：**iztro**（npm，MIT，浏览器内毫秒级排盘，含大限/流年）。

**数据流**：浏览器 iztro 排盘 → chart_json 存入档案 → 对话时前端发送（用户消息 + 盘面 + 镜头宫位 + 场景）→ 后端组装上下文 → Claude 流式回复 + 镜头指令 → 对话结束异步蒸馏记忆。

## 5. 数据模型（4 张新表）

| 表 | 关键字段 | 说明 |
|---|---|---|
| `ZiweiProfile` | name, relation, gender, birth_solar, birth_time_index, is_lunar_input, chart_json, persona, portrait_json | 命主档案；portrait_json 为 AI 蒸馏画像（性格关键词、关注主题、已验证事件摘要） |
| `ZiweiConversation` | profile_id, scenario(natal/horoscope/synastry/report), title, created_at | 会话，按档案隔离 |
| `ZiweiMessage` | conversation_id, role, content, chart_context_json | 消息 + 当时盘面上下文（流年年份、镜头宫位） |
| `ZiweiLifeEvent` | profile_id, event_date, category(事业/感情/健康/财务/其他), description, verified_against_json | 人生事件；与流年推演对照标注应验情况 |

合盘配对不建新表：合盘会话的 `chart_context_json` 存两个 profile_id 与合参结果。

## 6. AI 解盘循环（ziwei_oracle.py）

每轮请求组装 system prompt：

1. **人设 prompt**（personas.json 三选一，档案级偏好）
2. **命盘摘要**（chart_json → 紧凑文本：十二宫主星、亮度、四化、五行局、大限）
3. **命中格局知识**：规则引擎扫描 chart_json（星曜+宫位+四化组合精确判定），注入命中条目的格局释义与古籍选段——格局是规则可判定的，精确匹配优于语义检索，故不引入向量库
4. **档案画像** portrait_json + 最近 20 条人生事件
5. **场景指令**（本命盘/流年/合盘各自的解读框架）

工具：镜头指令集（§3.2）+ `record_life_event`（用户提到真实事件时记录）。多轮上下文保留最近 10 条消息（同 copilot 模式）。回复流式（SSE，同现有模式）。

**记忆蒸馏**：会话结束（或每 10 轮）后台异步调 Claude，把对话要点合并进 portrait_json，提及的真实事件写入 ZiweiLifeEvent。下次对话自动注入——越用越懂命主。

**预算**：纳入门户现有每日 token 预算管控。

## 7. 知识库构建（一次性离线脚本）

clone Renhuai123/ziwei-doushu → 提取脚本转换为 5 个 JSON：

- `patterns.json`：1100+ 行格局规则库 →（条件：星曜/宫位/四化组合，内容：释义+吉凶+建议）
- `stars.json`：十四主星 + 辅星佐星释义（含各宫位表现）
- `sihua.json`：四化飞星释义
- `classics.json`：骨髓赋等古籍选段（按格局/星曜索引）
- `personas.json`：三种人设 system prompt

总量约几百 KB，进版本库；文件头与页面页脚标注数据来源（MIT 要求）。

## 8. 四大场景

1. **本命盘**：建档后首次进入自动播放「开盘」运镜（可跳过），AI 主动给出开场总评（命宫格局 + 三方四正概览），用户自由点宫探索或提问。
2. **AI 对话**：贯穿所有场景的对话坞；支持针对任意宫位/星曜/格局追问，长期记忆生效。
3. **流年**：底部时间轴拖动切换年份；3D 盘面叠加流年层（流年宫位光环 + 流年四化飞光变化）；AI 按「大限定基调 → 流年看应期」框架解读；可与人生事件对照（「2023 年你换了工作，正应官禄宫化禄」）。
4. **合盘**：选两个档案，两盘并列悬浮、镜面相对，关键宫位（夫妻/交友/事业）之间拉出关系光线；AI 给契合度分析。
5. **报告**：深度解读沉淀为结构化报告（Markdown 存档），列表回看。

## 9. 错误处理与降级

- **排盘失败**（极端生辰/时区问题）：友好提示并指引修正生辰输入。
- **WebGL 不可用**：降级为精美 2D 命盘（SVG 方盘），AI 功能完整保留，镜头指令降级为高亮滚动。
- **Claude 超时/超预算**：对话坞内联错误 + 重试按钮；超预算显示明日恢复提示。
- **3D 性能**：进入页面先渲染星空 + 骨架；档位自适应（检测帧率自动降粒子/关 Bloom）；动画全部可跳过。
- **数据安全**：生辰属敏感个人信息，仅存自家库；删除档案级联删除全部记忆与对话。

## 10. 测试策略

- **排盘正确性**：用公认名盘快照测试 iztro 输出（十二宫安星、五行局、四化、大限起讫）。
- **格局规则引擎**：每条规则单测（命中/不命中用例）。
- **oracle 上下文组装**：单测验证 prompt 含命中格局、画像、事件。
- **API**：档案 CRUD / 对话 / 记忆蒸馏的集成测试，沿用现有 pytest 模式。
- **前端 3D**：Playwright 截图冒烟（场景加载、点宫飞入、降级路径）。
