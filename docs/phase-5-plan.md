# Phase 5 Plan

## Goal

Turn the portal into an adaptive learning system that reacts to actual progress quality, not only completion counts.

## Definition of Done

Phase 5 is complete when the portal can:

- identify weak areas across learning, practice, interview reps, and project proof
- recommend the next best lesson, drill, or interview move from that weakness
- show an interpretable mastery view instead of only aggregate progress
- adapt recommendations as activity changes

## Batch Order

### Batch A: Mastery Foundation

- introduce a mastery profile per core learning area
- blend lesson progress, drill reps, interview reps, and signal demand
- surface an adaptive focus card on the dashboard

### Batch B: Recommendation Re-ranking

- make recommendations favor the weakest meaningful area
- reduce repetition when an area is already stable
- improve practice rotation from mastery gaps rather than broad category counts

### Batch C: Mastery-aware Interview and Project Coaching

- connect mastery gaps to interview categories and project proof more explicitly
- show which project or interview rep best closes each weak area

### Batch D: Phase 5 Checkpoint

- verify adaptive behavior with live activity changes
- document the checkpoint and known limits

## Working Principle

Adaptive behavior should stay legible. The portal should feel coach-like, not random.
